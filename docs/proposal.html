The Title Shot

Going Live, Same URL

Three ways to put the new site on whitecornerboxing.in. They're ordered by what I'd actually recommend.

Recommended

Drop it onto Hostinger, keep WordPress underneath

The new site pulls its images and reel from whitecornerboxing.in/wp-content/…, so keeping WordPress on the same host means every asset URL keeps working and nothing breaks. Fastest, zero DNS changes, fully reversible.

  1. In hPanel → File Manager, open public_html and back up the current index.php (rename to index.php.bak).
  2. Upload index.html, robots.txt and sitemap.xml into public_html. LiteSpeed serves index.html before WordPress's index.php.
  3. Keep the whole wp-content/ folder in place — that's where the videos and photos live. WordPress admin stays reachable at /wp-admin for the blog & events.
  4. Point old page URLs (/about, /gallery) either at the new one-pager's sections or leave them on WordPress — your call.
  5. In Google Search Console, submit the new sitemap.xml and request re-indexing of the homepage.
  6. To roll back: delete index.html, restore index.php. Instant.
Why this one: same host, same domain, same assets, WordPress blog/events preserved, and reversible in seconds. It's the lowest-risk path to the same URL.
Alternative

Cloudflare Pages / Netlify, point the domain over

Free, global CDN, automatic HTTPS, git-based deploys. Best long-term performance — but because it leaves Hostinger, you must first move the media the site depends on.

  1. Copy wp-content/uploads (the videos + photos) into the project so nothing hotlinks to the old host, or keep WordPress running purely as a media store.
  2. Push index.html + assets to a GitHub repo; connect it to Cloudflare Pages.
  3. In Hostinger DNS, change the root A/CNAME record to point at the Pages project.
  4. Add whitecornerboxing.in as a custom domain in Pages; HTTPS is issued automatically.
  5. Decide what happens to the WordPress blog — subdomain (blog.whitecornerboxing.in) or retire it.
Alternative

Staging subdomain first

Lowest-commitment way to see it on real infrastructure before touching the live homepage.

  1. Create new.whitecornerboxing.in as a subdomain in hPanel.
  2. Upload the three files there and review on real hosting, on phones.
  3. When approved, promote it to the root using the recommended route above.
Two things to settle before launch: (1) the new site currently loads images/video from the live domain — on the recommended route that's fine; on the Cloudflare route the media must move first. (2) Re-point the existing GA4 + GTM tags into the new page so analytics don't go dark.
